Iteration 2
Ask the user whether they would like a beverage (yes, no).
If they say yes, ask what size beverage they would like:
small $1.00, medium $1.75, large $2.25
When they select a size, ask what type of beverage they would like (Sweet tea, Coca-Cola, Root Beer, or Water)
Have the program output the user’s beverage selection, size selection, or total lack of selection, to verify that the program is working correctly.
Have the program output the total cost so far.
Iteration 3
Ask the user whether they would like chips (yes, no).
If they say yes, ask what size bag of chips they would like: small $1.00, medium $1.50, large $2.00
If they say “small,” ask the user whether they’d like to mega-size their chips (yes, no).
If the user inputs yes to mega-size, give them large bag of chips at the large chips price instead of their small bag of chips and small bag of chips price.
Have the program output the user’s chip selection to verify that the program is working correctly.
Adjust the program so the total cost only outputs to the user after their taco, drink, and chips selection.
Iteration 4
Ask the user how many sauce packets they would like (enter a positive integer; cost is $0.25 per packet).
After ordering the taco, drink, chips, and sauce packets:
If the user selected a taco, chips, and a beverage, reduce the total cost of the order by $1.00.
The program informs the user of their menu selections, for only the items they ordered.
The program should print the total cost of the order. Remove any other totals before this point.
